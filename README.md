# product-owner-repo

## Purpose
This repository is the system of record for an Product Owner responsible for:

- PI roadmap creation
- Feature/Epic and Story backlog management
- Development story creation with acceptance criteria
- Defect and emergency backlog management

Jira remains the system of execution.
This repository documents structure, intent, prioritization, and decisions.

## What Lives Here
- PI planning intent and assumptions
- Backlog hierarchy (Epics → Features → Stories)
- Acceptance criteria standards
- Defect and emergency handling rules
- Scope and priority trade-off decisions

## What Does NOT Live Here
- Sprint boards
- Estimates or capacity tracking
- Daily execution status

## Operating Model
- Jira = execution
- GitHub = planning clarity and decision memory
